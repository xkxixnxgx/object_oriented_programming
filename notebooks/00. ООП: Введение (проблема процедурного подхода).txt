# ООП решает проблему поддержки кода, который необходимо часто менять, для чего процедурный код не очень подходит.
# ООП позволяет скрыть внутреннюю релизацию и выдает наружу строго ограниченный интерфейсю
# ООП позволяет переиспользовать модули
# ООП требует тщательной подготовки классов, их взаимодействия, необходимо продумать систему прежде, чем начинать писать классы..
# Если для решения задачи можно обойтись без ООП, лучще ООП не использовать.
